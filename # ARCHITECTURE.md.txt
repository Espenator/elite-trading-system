### 2. Full Trade Pipeline

```
AlpacaStreamManager (multi-key WebSocket)
    │ market_data.bar
    ▼
EventDrivenSignalEngine (DuckDB queries + technical analysis)
    │ signal.generated (score 0-100)
    ▼
CouncilGate (threshold ≥ 65, max_concurrent=3, cooldown=120s)
    │ invokes run_council()
    ▼
┌─────────────────────────────────────────────┐
│        35-AGENT DAG (7 Stages)              │
│                                              │
│ Stage 1: 13 agents (perception + P0-P2)     │
│ Stage 2: 8 agents (technical + enrichment)  │
│ Stage 3: 2 agents (hypothesis + memory)     │
│ Stage 4: 1 agent (strategy)                 │
│ Stage 5: 3 agents (risk + exec + portfolio) │
│ Stage 5.5: 3 agents (debate + red team)     │
│ Stage 6: 1 agent (critic)                   │
│ Stage 7: arbiter (deterministic BUY/SELL/HOLD)│
│ Post: 1 agent (alt_data, background)        │
└─────────────┬───────────────────────────────┘
              │ council.verdict
              ▼
OrderExecutor (real DuckDB stats, ATR, Kelly sizing, mock-source guard)
    │ order.submitted
    ▼
Alpaca Markets API (paper/live)
    │
    ▼
WebSocket bridges → Frontend Dashboard
```

### 6. Brain Service

- **Location:** `brain_service/` (7 files + proto/ directory)
- **gRPC schema:** `brain_service/proto/brain.proto`
- **Server:** `brain_service/server.py` — gRPC server on port 50051
- **Client:** `brain_service/ollama_client.py` — calls Ollama API
- **Model:** llama3.2 (configurable via .env)
- **Wiring:** `hypothesis_agent.py` calls brain gRPC via `app/services/brain_client.py` [github](https://github.com/Espenator/elite-trading-system/blob/main/backend/app/main.py)

### 8. DuckDB Schema

Primary tables (from data/duckdb_storage.py):
- `daily_ohlcv` — Symbol, date, OHLCV, source
- `features` — Computed feature vectors (50+ indicators)
- `signals` — Generated trading signals
- `trades` — Executed trade records
- `trade_stats` — Win rate, avg win/loss aggregates
- `model_artifacts` — ML model metadata
- `patterns` — Detected chart patterns
- `settings` — User settings key-value store

### 10. Dual-PC Hardware Topology

- **ESPENMAIN (PC1):** Development + Backend API + Frontend + DuckDB + Alpaca streaming
- **PC2 (ProfitTrader):** RTX GPU + brain_service (gRPC :50051) + Ollama inference
- **Connection:** gRPC over LAN, port 50051, configured via `CLUSTER_PC2_HOST` env var
- **NodeDiscovery:** Automatic PC2 health monitoring with tiered fallback
