# Running Embodier Trader 24/7 with Auto-Restart

Backend, frontend, and (optionally) Electron restart automatically when they crash or stop responding. API and WebSocket URLs are auto-fixed so the dashboard and real-time updates work. Port clearing runs before start when using `-CleanPorts` or the 24/7 auto launcher. For automatic start each day at logon, use `.\scripts\install-task-scheduler.ps1 -FullStack247`.

## Automatic start each day / at logon (recommended for 24/7)

To have the system **start automatically** when you log in and stay running 24/7 with port clearing, auto-restart, and API/WebSocket fixed:

```powershell
.\scripts\install-task-scheduler.ps1 -FullStack247
```

- Creates a scheduled task **EmbodierTrader-24-7** that runs at **logon** (with a 30s delay).
- Runs `scripts\start_embodier_24_7_auto.ps1`: port cleanup, writes `frontend-v2/.env` (VITE_PORT, VITE_BACKEND_URL, VITE_WS_URL), then starts backend + frontend with **supervisor** (auto-restart on crash). Electron is skipped by default.
- **One** supervisor window stays open; backend and frontend restart automatically if they exit or fail health checks.
- To enable **auto-execute** (live trading) at startup: set `AUTO_EXECUTE_TRADES=true` in `backend/.env`, or run once: `.\scripts\start_embodier_24_7_auto.ps1 -EnableAutoExecute`

To remove: `.\scripts\install-task-scheduler.ps1 -Uninstall`

## One launcher only — avoid duplicate windows

**Use only ONE of the options below.** Do not run multiple launchers at once (e.g. Task Scheduler + `run_full_stack_24_7.ps1`, or `start-monitor.ps1` + `run_all_autorestart.ps1`). That causes extra windows and can look like "a new window every few seconds" if components keep restarting.

- **If you use `run_full_stack_24_7.ps1` or `run_all_autorestart.ps1` or `start-embodier.ps1 -Watch`:**  
  Disable any auto-start so it doesn’t start again at logon:
  - **Task Scheduler:** `.\scripts\install-task-scheduler.ps1 -Uninstall` or delete task `EmbodierTrader` / `EmbodierTrader-24-7`.
  - **Startup shortcut:** Remove `Embodier Trader.lnk` from `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`.
- **If you use Task Scheduler or Startup to run `start-all.ps1` or `start-monitor.ps1`:**  
  Don’t also run `run_full_stack_24_7.ps1` or `run_all_autorestart.ps1` in another window.

The backend autorestart script no longer opens a new console window on each restart (uvicorn runs hidden). If you need to see backend output, run:  
`.\backend\scripts\run_backend_autorestart.ps1 -Port 8000 -ShowBackendWindow`

## Default: app runs and restarts automatically (hardcoded)

- **Desktop shortcut**: Run once `.\scripts\create-desktop-shortcut.ps1` to create **"Embodier Trader"** on your desktop. Double-click it to stop existing processes, clear ports, and start backend + frontend with supervisor and **auto-execute** (no manual steps).
- **`.\start-embodier.ps1`** (no args): Stops any existing processes, then starts the 24/7 stack (CleanPorts + Supervisor, NoElectron, **EnableAutoExecute**). One supervisor window stays open; backend and frontend auto-restart.
- **`.\scripts\run_full_stack_24_7.ps1`** (no args): CleanPorts, Supervisor, NoElectron, **EnableAutoExecute** by default. Port clearing always uses 8000/5173 when cleaning (no stale ports from file). Use `-NoEnableAutoExecute` to disable auto-execute; `-NoCleanPorts -NoSupervisor` for one-shot.
- **Frontend** hardcodes API `http://localhost:8000` and WS `ws://localhost:8000` when env vars are not set, so the app runs and reconnects without a `.env` file.

## Get the app running now (API/WS down or DuckDB lock)

If the API or WebSocket are down, or you see "DuckDB file in use", **only one backend** can hold the DB. Do this:

1. **Stop everything** (from repo root):
   ```powershell
   .\scripts\stop_embodier.ps1
   ```
2. **Start** (default is 24/7 with auto-restart):
   ```powershell
   .\start-embodier.ps1
   ```
   Or directly: `.\scripts\run_full_stack_24_7.ps1`
3. Keep the **supervisor window** open. Dashboard: `http://localhost:<frontendPort>/dashboard` (port in `.embodier-ports.json`).

## Quick start (recommended)

**Option A – One-shot 24/7 launcher (port cleanup + API/WS fix + supervisor)**  
From repo root (ESPENMAIN PC1):

```powershell
.\scripts\start_embodier_24_7_auto.ps1
```

- Runs port cleanup, updates `frontend-v2/.env` (VITE_PORT, VITE_BACKEND_URL, **VITE_WS_URL**), then starts full stack with supervisor. Optional: `-NoElectron`, `-EnableAutoExecute`.

**Option A1 – Full stack 24/7 with one supervisor window (manual)**  
```powershell
.\scripts\run_full_stack_24_7.ps1 -Supervisor -CleanPorts
```

- **One** supervisor window stays open; it starts backend, frontend, and Electron in separate windows and only restarts them when they exit. No extra windows every few seconds.
- Backend runs with hidden console (no new window on each backend restart). Frontend .env gets VITE_WS_URL so WebSocket works.
- Use `-NoElectron` to skip the desktop app. Use `-EnableAutoExecute` to enable trading via API after backend is ready. Ports written to `.embodier-ports.json`.

**Option A2 – Full stack without supervisor (three windows, each restarts itself)**  
```powershell
.\scripts\run_full_stack_24_7.ps1 -CleanPorts
```
- Starts backend, frontend, and Electron in three separate windows; each auto-restarts on exit or (backend) health failure.

**Option B – Full launcher (cleanup + port resolution + watch, backend + frontend only)**  
From repo root:

```powershell
.\start-embodier.ps1 -Watch
```

- Cleans stale Python/Node processes and frees ports.
- Resolves backend (8000–8010) and frontend (5173–5183) if preferred ports are busy.
- Starts backend in one window (auto-restart + health watchdog every 30s).
- Waits for backend `/health` (max 90s), then starts frontend in another window (auto-restart on exit).
- Press Enter in the launcher window to stop both.

**Option C – Auto-restart only (use existing or default ports)**  
From repo root:

```powershell
.\scripts\run_all_autorestart.ps1 -CleanPorts
```

- `-CleanPorts`: kills processes on 8000/5173 (or next free in range), writes `.embodier-ports.json`.
- Starts backend and frontend in separate windows; each restarts on failure.
- Waits for backend health before starting frontend so the dashboard gets API data immediately.

Without `-CleanPorts`, uses ports from `.embodier-ports.json` if present, else 8000/5173.

**Electron only (backend + frontend already running)**  
From repo root:

```powershell
.\desktop\scripts\run_electron_autorestart.ps1
```

- Restarts the Electron desktop app when it exits; uses existing backend/frontend ports from `.embodier-ports.json` or defaults.

## What keeps running

| Component   | Window / process      | Restart trigger                          |
|------------|------------------------|------------------------------------------|
| Backend    | 1 PowerShell window; uvicorn runs **hidden** (no extra console per restart) | Process exit or 3× failed `/health` (30s interval) |
| Frontend   | PowerShell + Vite     | Process exit (`npm run dev` stops)       |
| Electron   | PowerShell + Electron| Process exit (app closed or crash)       |

- **Backend**: `backend/scripts/run_backend_autorestart.ps1` – sets `PORT`, pings `http://127.0.0.1:PORT/health` every 30s; after 3 consecutive failures (after 45s startup grace), kills and restarts. Uvicorn is started with `-WindowStyle Hidden` so each restart does **not** open a new visible window. Restart events logged to `logs/backend_autorestart.log` when present.
- **Frontend**: `frontend-v2/scripts/run_frontend_autorestart.ps1` – sets `VITE_PORT` and `VITE_BACKEND_URL` so the dev server and proxy target the correct backend; restarts after 2s when Vite exits.
- **Electron**: `desktop/scripts/run_electron_autorestart.ps1` – runs `npm run start` in a loop; restarts after 5s when the app exits.

## API and WebSocket (auto-fixed)

- All 24/7 and watch scripts write **frontend-v2/.env** with `VITE_PORT`, `VITE_BACKEND_URL`, and **VITE_WS_URL** so the dashboard and real-time WebSocket connect to the correct backend.
- Frontend uses **Vite proxy**: `/api` → backend, `/ws` → backend WebSocket. Backend URL and WS URL come from the env vars set by the scripts.
- Backend binds to `0.0.0.0:PORT` so it accepts requests from the frontend and other hosts.
- Ports are stored in `.embodier-ports.json` and `frontend-v2/.env` so manual runs of `npm run dev` or `uvicorn` can use the same ports.

## If something drops

1. **Backend dies or hangs**  
   Watchdog detects failed health checks and restarts the backend process in the same window. Frontend will show “API OFFLINE” until `/health` returns 200 again.

2. **Frontend dies**  
   The frontend window’s loop restarts `npm run dev` after 2 seconds. Reload the browser if needed.

3. **Ports stuck (e.g. TIME_WAIT)**  
   Run with port resolution so the next free port is used:
   - `.\start-embodier.ps1 -Watch` (always resolves), or  
   - `.\scripts\run_all_autorestart.ps1 -CleanPorts`

## URLs

- **Dashboard**: `http://localhost:<frontendPort>/dashboard`
- **Backend API docs**: `http://localhost:<backendPort>/docs`
- **Health**: `http://localhost:<backendPort>/health`

Ports are printed when the launcher starts and (when applicable) saved in `.embodier-ports.json`.

## Stop duplicate / annoying auto-start (new window every few seconds)

If you see new PowerShell or console windows opening every few seconds:

1. **Use only one 24/7 launcher** (see "One launcher only" above). Prefer automatic at logon:
   ```powershell
   .\scripts\install-task-scheduler.ps1 -FullStack247
   ```
   Or run manually: `.\scripts\start_embodier_24_7_auto.ps1` or `.\scripts\run_full_stack_24_7.ps1 -Supervisor -CleanPorts`. That keeps one supervisor window open and restarts backend/frontend/Electron only when they actually exit.

2. **Remove duplicate auto-start:**
   ```powershell
   .\scripts\install-task-scheduler.ps1 -Uninstall
   ```
   Then delete the Startup shortcut if present:  
   `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\EmbodierTrader.lnk`

3. **Don’t run** `start-monitor.ps1` or `start-all.ps1` in the background if you are already using `run_full_stack_24_7.ps1` or `run_all_autorestart.ps1`.
