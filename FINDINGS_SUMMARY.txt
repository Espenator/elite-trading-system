CRITICAL FINDINGS:
==================

1. HARDCODED 0.4 EXECUTION THRESHOLD (Line 152, arbiter.py)
   - Threshold is NOT regime-adaptive
   - In CRISIS (VIX>50): 40% confidence is dangerously reckless
   - In BULLISH (VIX<12): 40% is overly conservative
   - Should scale from 0.30 (bullish) to 0.70 (crisis)
   - FIX: Implement regime_thresholds dict in arbiter.arbitrate()

2. NO OVERALL COUNCIL TIMEOUT (runner.py line 42+)
   - Individual agents timeout after 30s (task_spawner.py:204)
   - But entire council DAG has no wall-clock limit
   - Debate stage can run indefinitely
   - Could cause councils to hang >5 minutes under load
   - FIX: Add asyncio.wait_for(..., timeout=5.0) wrapper around run_council()

3. AGENT FAILURES SILENTLY CONVERT TO HOLD VOTES (task_spawner.py:217-236)
   - All timeouts/exceptions become HOLD votes
   - No health tracking or escalation
   - If 50% of agents fail, council proceeds with partial votes
   - No visibility into system degradation
   - FIX: Track agent health, halt council if >30% failure rate

   - Long signals use [0, +25] (symmetric)
   - Biases against short signals
   - FIX: Change to max(0, min(25, ...)) for symmetry

7. OpenClaw Blending Weights Fixed (60%) Regardless of Quality
   - signal_engine.py line 454: blended = (ta_score * 0.4) + (pillar_score * 0.6)

    - signal_engine.py lines 616-657
    - Both long and short signals can fire in same bar

    - FIX: If long fires, suppress short evaluation
